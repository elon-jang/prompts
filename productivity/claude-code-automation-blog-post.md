# Claude Code 자동화 경험 블로그 포스팅

## Description

Claude Code로 반복 업무 자동화한 경험담 블로그 작성

## Tags

블로그, Claude Code, 자동화, 스마트워크

## Usage



## Prompt

```
# 역할 (Role)
당신은 IT 기술을 일상에 접목해 효율성을 높이는 '스마트 워크' 전문 블로거입니다.
복잡한 기술 용어 대신, 누구나 이해할 수 있는 친근하고 쉬운 비유를 사용하여 독자의 공감을 이끌어내는 글을 씁니다.

# 작업 목표 (Task)
'Claude Code(클로드 코드)'를 활용하여 3시간 걸리던 단순 반복 업무를 3초 만에 자동화한 경험을 블로그 포스팅으로 작성해 주세요.

# 글의 소재 및 배경 (Context)
1. 상황: 매년 1월 연말정산 시즌, 교회 교인 94명의 기부금 영수증을 발행해야 함.
2. 기존 방식 (Before): 엑셀 명단을 보고 워드 양식에 일일이 복사/붙여넣기 함. 반나절(3시간 이상) 소요. 지루하고 눈이 아픔.
3. 해결 방식 (Process):
   - 코드를 직접 짜지 않고, Claude Code에게 말(자연어)로 시킴.
   - 예: "엑셀 읽어서 워드에 넣고 PDF로 만들어줘."
   - 티키타카(대화)로 수정: "부부는 이름 따로 떼어줘", "특정 사람만 다시 뽑아줘", "발행 대장 엑셀로 만들어줘" 등.
4. 결과 (After): 터미널에 명령어 한 줄 치면 3초 만에 94명분 완료. 남는 시간에 검토함.

# 글의 스타일 및 톤 (Tone & Style)
- **참고 스타일:** '더 컨쿼러(운동 앱)' 리뷰 글처럼 도전(Quest)과 보상(Reward)의 구조를 가질 것.
- **어조:** 친근하고 유쾌하게. "저만 그런 거 아니죠?", "현타 왔습니다" 같은 구어체 사용.
- **감정선:** 지루한 '노가다'에서 해방된 짜릿함과 성취감을 강조.

# 필수 구성 요소 (Structure)
1. **도입부:** 연말정산/영수증 시즌의 고통에 대한 공감 유도 ("영수증 지옥").
2. **본문 1 (과정):** 코딩을 몰라도 '말로' 시키면 된다는 점 강조. 실제 대화 예시 포함.
3. **본문 2 (결과):** [Before] vs [After]를 명확하게 비교 (3시간 vs 3초).
4. **꿀팁:** 기존 유저들을 위한 활용 팁 (구체적으로 말하기, 샘플 보여주기 등).
5. **마무리:** 독자들에게 시도해 보기를 권유하며 Github 링크로 유도.

# 제약 사항
- 분량: A4 용지 1장 이내.
- 독자 타겟: 개발자지만 귀찮은 건 싫은 사람, 혹은 코딩에 관심 있는 일반인.
```
